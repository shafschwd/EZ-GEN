## The "capacitor.settings.gradle not found" Error

This is a common issue that occurs when Capacitor hasn't been properly synced after app generation.

### Why This Happens

1. The `capacitor.settings.gradle` file is auto-generated by Capacitor
2. It contains references to all Capacitor plugins and the core Android module
3. Without this file, Gradle cannot find the required dependencies

### How EZ-GEN Prevents This

Starting from this version, EZ-GEN automatically:

1. âœ… Creates a placeholder `capacitor.settings.gradle` file in the template
2. âœ… Runs `npm install` to install all dependencies
3. âœ… Runs `npm run build` to create the web assets
4. âœ… Runs `npx cap sync` to generate proper Capacitor files
5. âœ… Provides setup scripts for manual fixing if needed

### Manual Fix (If Needed)

If you still encounter this error:

#### Quick Fix
```bash
cd your-app-directory
npx cap sync
```

#### Complete Fix
```bash
cd your-app-directory
npm install
npm run build
npx cap sync
```

#### Using the Setup Scripts

**Windows:**
```bash
path\to\ez-gen\setup-app.bat
```

**Mac/Linux:**
```bash
node path/to/ez-gen/setup-app.js
```

### What Gets Generated

After `npx cap sync`, the `capacitor.settings.gradle` file will contain:

```groovy
// DO NOT EDIT THIS FILE! IT IS GENERATED EACH TIME "capacitor update" IS RUN
include ':capacitor-android'
project(':capacitor-android').projectDir = new File('../node_modules/@capacitor/android/capacitor')

include ':capacitor-app'
project(':capacitor-app').projectDir = new File('../node_modules/@capacitor/app/android')

// ... other plugins
```

### Prevention for Future Apps

The EZ-GEN server now automatically handles this during app generation, so new apps should work immediately without manual intervention.
